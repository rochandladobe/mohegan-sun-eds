# Mohegan Sun EDS — Homepage Authoring Structure

This document maps each homepage section to its corresponding EDS block and Google Doc / DA authoring structure. Use this as a reference when building or updating the content document at [da.live](https://da.live/#/rochandladobe/mohegan-sun-eds).

---

## Overview

The EDS homepage (`/`) is authored as a single Google Doc or DA document. Each section is represented by a **block table** (a 2-column or multi-column table whose first cell contains the block name). Sections are separated by horizontal rules (`---`) which become new `<div>` wrappers in the HTML output.

### HTML output structure (per slide)

```html
<div class="hero-carousel">
  <div><div>
    <picture><img src="..." alt="..."/></picture>
    <h2>Slide Headline</h2>
    <p>Slide subtitle</p>
    <p><strong><a href="/path">CTA Label</a></strong></p>

```

**Notes:**
- Images should be 1440×620px.
- CTA link wrapped in `**bold**` renders as `<strong>` in EDS, which the block JS uses to style the primary button.
- The block JS (`/blocks/hero-carousel/hero-carousel.js`) handles auto-advance, dot indicators, and swipe support.

## Section Metadata Reference

`Section Metadata` is a special EDS block that applies attributes to its containing section `<div>`. Place it as the **first block** in any section that needs a visual treatment.

| Style value | Effect |
|---|---|
| `dark` | Dark background (near-black), light text — used for entertainment and online gaming banners |
| `purple` | Deep brand purple background, light text — used for stats bar |
| *(none)* | Default white/light background |
